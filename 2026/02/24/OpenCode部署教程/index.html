这篇内容是面向实战的 OpenCode 部署指南：先把 CLI 跑起来，再逐步接入桌面版、插件版和常见模型供应商。

1. 四种使用形态

OpenCode 常见的使用方式有四种：

  1. 命令行（CLI）
  2. 桌面客户端（Desktop）
  3. 编辑器插件（例如 VS Code）
  4. 云端运行环境（Cloud）

建议顺序是：先 CLI，再插件/桌面，最后补云端。

3. 安装桌面版（可选）

  1. 下载对应系统安装包并安装。
  2. 首次打开后选择项目目录。
  3. 进入模型列表确认 Provider 已加载。

桌面版更适合轻量对话与快速体验；复杂工程任务依然建议以 CLI 为主。

4. 安装 VS Code 插件版

  1. 在扩展市场搜索并安装 OpenCode。
  2. 按 Ctrl + Shift + P 打开命令面板，执行 OpenCode 相关命令。
  3. 选中代码后可直接发送到 OpenCode 对话窗口，减少复制粘贴。

插件版依赖本地 OpenCode 环境，所以请先完成 CLI 安装。

7. 接入 Codex（OpenAI）

  1. 执行 /connect。
  2. 选择 OpenAI 并完成浏览器授权。
  3. 执行 /models 切换到目标编码模型。

完成后就可以在 OpenCode 里直接使用对应模型进行编码任务。

8. 接入 OpenRouter

  1. 在 OpenRouter 控制台申请 API Key。
  2. 在 OpenCode 执行 /connect，选择 OpenRouter 并粘贴密钥。
  3. 执行 /models 验证模型列表。

9. 推荐部署顺序（避免折腾）

  1. 先完成 CLI 安装与版本验证。
  2. 接入一个主力 Provider（OpenAI 或 OpenRouter）。
  3. 再补充 Antigravity 等高级模型通道。
  4. 固定一个默认模型作为日常主模型。
  5. 最后接入 VS Code 插件提升效率。

10. Session 并行协作建议

把不同任务拆成多个 Session：

  • Session A：新功能开发
  • Session B：Bug 修复
  • Session C：重构/性能优化

这样能减少上下文干扰，也更方便回溯。

11. 常见问题

Q1：opencode 提示命令不存在

通常是 npm 全局目录未加入 PATH。重开终端后重试；不行就检查 Node/npm 的全局安装路径。

Q2：/models 看不到刚接入的模型

检查三项：授权是否完成、Provider 是否连接成功、会话是否已重启。

Q3：应该优先桌面版还是 CLI？

优先 CLI。桌面版与插件版更适合作为辅助入口。
